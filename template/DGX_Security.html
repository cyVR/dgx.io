Security Features

We take measures to ensure security and go beyond “bank-level security” standards to protect our customers. Here are some of the technical and operational measures we have in place.

Technical

  • Our site runs entirely over SSL and enforce strict transport security (HSTS)
  • We use Content Security Policy as a way to prevent cross-site scripting attacks (XSS)
  • We sanitize all user input and have proper SQL injection filters
  • We use cross-site request forgery (CSRF) authentication measures.
  • We enforce 2-Factor authentication for all accounts.
  • We use a combination of hardware random number generators (HRNG) and cryptographically secure pseudo-random number generators (CSPRNG) for key generation.
  • We have an air-gapped application and network infrastructure.
  • Login sessions are expired using a strict set of rules to prevent session fixation attacks.
  • Our web application enforces resource access policies and strong parameters to prevent mass-assignment attacks.
  • Sensitive user information is encrypted on the database using public key cryptography. The private keys are kept in separate servers only accessible from our internal network.
  • We enforce per-user access control on our database tables.
  • Our servers run security hardened kernel using GRSecurity to defend against zero-day vulnerabilities and to enforce role based access control.
  • Our password policy requires passwords with high entropy.
  • Passwords are expired every 3 months and users may not use the same password again.
  • We use different passwords for every server and application.
  • Support and administrative dashboards can only be accessed from company workstations which are assigned client SSL certificates as part of the 3 step authentication process.
  • Inbound and outbound internet traffic from our office network is strictly controlled.
  • Any external libraries used in our application are audited and go through a strict approval process.
  • Unauthorized attempts and other anomalous activities are logged.
  • Remote employees can only connect to our internal network through a private VPN connection using a company issued laptop.
  • Our IT infrastructure and applications are designed, developed, and implemented with security as a primary concern rather than an afterthought.

Operational Security (OpSec)

  • Employees are required to pass a criminal background check.
  • Employees are assigned company controlled smart phones which are remotely wiped if lost or stolen.
  • Employees are trained and educated about the risks and methods of social engineering.
  • All sensitive internal communications are conducted over secure channels.
  • We enforce strict separation of duties.
  • All account management and support activities go through an approval process.
  • We respond to all security alerts with proper escalation procedures.
